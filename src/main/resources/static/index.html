Алгоритмы сортировки и поиска (Java) — Теоретическая часть

1. Цель работы

Изучить принципы работы различных алгоритмов сортировки и поиска, реализовать их на языке Java, провести сравнительный анализ времени выполнения сортировки на массивах разного размера и сделать выводы об эффективности алгоритмов.

2. Задачи

  • Изучить теоретические основы алгоритмов сортировки (пузырьковая, вставками, выбором, слиянием, быстрая) и алгоритмов поиска (линейный, бинарный).
  • Реализовать данные алгоритмы на Java.
  • Сгенерировать тестовые массивы размером 100, 1000 и 5000 элементов, заполненных случайными целыми числами.
  • Измерить время выполнения каждого алгоритма сортировки на полученных массивах.
  • Составить таблицу сравнения времени работы.
  • Проанализировать результаты и сформулировать выводы.

3. Теоретическая часть

3.1 Алгоритмы сортировки

Сортировка пузырьком (Bubble Sort) — простой алгоритм, который многократно проходит по массиву, попарно сравнивает соседние элементы и меняет их местами, если они идут в неправильном порядке. После каждого прохода «тяжёлые» элементы «всплывают» в конец массива.

  • Худшее и среднее время: O(n²)
  • Лучшее время (почти отсортированный массив): O(n)
  • Память: O(1)
  • Стабильный: да

Сортировка вставками (Insertion Sort) — строит отсортированную подпоследовательность, последовательно «вставляя» каждый следующий элемент на нужное место в уже отсортированной части массива. Хорошо работает на маленьких и почти отсортированных массивах.

  • Время: O(n²), лучшее — O(n)
  • Память: O(1)
  • Стабильный: да

Сортировка выбором (Selection Sort) — на каждом шаге находит минимальный элемент в неотсортированной части массива и обменивает его с первым элементом этой части. Количество сравнений почти не зависит от исходного порядка элементов.

  • Время: O(n²)
  • Память: O(1)
  • Стабильный: нет (в базовой реализации)

Сортировка слиянием (Merge Sort) — алгоритм «разделяй и властвуй»: рекурсивно делит массив пополам, сортирует каждую половину и затем сливает две отсортированные части в один массив.

  • Время (лучшее/среднее/худшее): O(n log n)
  • Дополнительная память: O(n)
  • Стабильный: да

Быстрая сортировка (Quick Sort) — также использует подход «разделяй и властвуй». Выбирается опорный элемент (pivot), массив делится на элементы меньше и больше опорного, затем эти части рекурсивно сортируются.

  • Среднее время: O(n log n)
  • Худшее время (неудачный выбор опорного): O(n²)
  • Память: O(log n) за счёт рекурсии
  • Стабильный: нет (в классической реализации)

Встроенная сортировка Arrays.sort в Java использует оптимизированные гибридные алгоритмы (например, TimSort для объектов и Dual-Pivot QuickSort для примитивов), обеспечивая высокую производительность на практике.

3.2 Алгоритмы поиска

Линейный поиск (Linear Search) — последовательно просматривает элементы массива до тех пор, пока не найдёт нужный или не достигнет конца.

  • Время: O(n)
  • Память: O(1)
  • Требования к данным: массив может быть не отсортирован

Бинарный поиск (Binary Search) — работает только на отсортированных массивах. На каждом шаге сравнивает искомый элемент с серединой массива и отбрасывает половину элементов, в которой элемент заведомо отсутствует.

  • Время: O(log n)
  • Память: O(1) в итеративной реализации
  • Требования к данным: массив должен быть отсортирован

4. Аналитические оценки сложности

Алгоритм Лучшее время Среднее время Худшее время Память
Bubble Sort O(n) O(n²) O(n²) O(1)
Insertion Sort O(n) O(n²) O(n²) O(1)
Selection Sort O(n²) O(n²) O(n²) O(1)
Merge Sort O(n log n) O(n log n) O(n log n) O(n)
Quick Sort O(n log n) O(n log n) O(n²) O(log n)
